## Ответы на вопросы

### 1. Какой корневой раздел программа пытается открыть
`HKEY_CURRENT_USER`

### 2. Какой подключ программа пытается открыть
`Software\78JziCwxMT`

### 3. Какой параметр подключа программа пытается открыть
`0xRMi`

### 4. Что произойдет, если программа не найдет такой подключ
Если `RegOpenKeyExA(HKEY_CURRENT_USER, "Software\\78JziCwxMT", ...)` вернет ошибку, программа:

1. Покажет окно:
   - заголовок: `NO !`
   - текст: `UoQ9uTWutWWl`
2. Вызовет `sub_4017BB()`
3. Завершится через `ExitProcess(0)`

### 5. Какое значение должен хранить параметр подключа, чтобы программа вывела `FLAG`
Параметр `0xRMi` должен хранить строку:

```
ZlkntxTQ29rPlTxTu8SWbd2nz1pOL99zJjCeWIIi
```
![alt text](/2.png)

### 6. Зачем дважды подряд вызывается WinAPI функция `RegQueryValueExA`

1. Первый вызов идет с `lpData = NULL`, чтобы узнать размер данных и записать его в `cbData`
2. Затем программа выделяет буфер нужного размера
3. Второй вызов уже читает само значение в этот буфер

## Вся логика:

Функция `sub_4017D2` реализует обычное `base64`-кодирование.

Программа делает следующее:

1. Читает значение параметра `0xRMi`
2. Кодирует его в `base64`
3. Сравнивает результат с заранее вычисленной строкой
4. Если строки совпадают, показывает окно `FLAG`

Ожидаемая строка сравнения:

```text
WmxrbnR4VFEyOXJQbFR4VHU4U1diZDJuejFwT0w5OXpKakNlV0lJaQ==
```

Ее декодирование из `base64` дает:

```
ZlkntxTQ29rPlTxTu8SWbd2nz1pOL99zJjCeWIIi
```

Именно это значение нужно записать в реестр.
