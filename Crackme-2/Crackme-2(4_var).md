## Проверка ключа:

`WinMain` - просто создает окно.

Интересное находится тут:

```
sub_401BC3
```

Это обработчик окна. Он следит за тем, что происходит в программе.

Когда нажимается кнопка `SUBMIT`, программа берет текст из поля ввода и передает его сюда:

```text
sub_4017BD
```

Вот эта функция и проверяет ключ.


Ключ должен выглядеть так:

```
XXXXXXXX-XXXXXXXX-XXXXXXXX-XXXXXXXX
```

Каждая часть - это число в hex.

## Как идет проверка

Программа берет эти четыре части как четыре числа:

```text
A-B-C-D
```

Последняя часть, то есть - `D`, должна быть:

```
00000A3A
```

`0xA3A` - это 2618. Это количество итераций.

Потом программа 2618 раз перемешивает первые три числа через сложение, вычитание и XOR.

После всех этих раундов должны получиться нужные значения:

```text
A = 7AA5F10B
B = 14994376
C = B9487CF2
D = 00000A3A
```

Если все совпало - ключ верный.

## Гибкость ключа:

В программе есть маленький глобальный буфер:

```
0x404010
```

Программа через него восстанавливает число:

```
0x94E
```

Это число участвует в проверке ключа.

Но загвоздка в том, что программа этот буфер меняет. Поэтому если сначала ввести неправильный ключ, то состояние внутри программы уже будет другим.
